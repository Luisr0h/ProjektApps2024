Cargar imágenes

El origen puede ser absoluto o relativo. En el caso de ser absoluto la img se encuentra en línea y se accede a ella a través de su link. En el caso de ser relativo. la encontraremos entre nuestros archivos locales. Se accede a ella a través de ./ en el caso de encontrarse dentro de la carpeta local, en caso de encontrarse en una carpeta de un dominio superior al del archivo, accedemos a la img a través de : .

Clicando en la imagen anterior, encontrarás una página que te ayuda a generar imágenes vacías con el formato especificado para así poder ir acomodando las imágenes en el espacio, aún sin tener las imágenes finales/definitivas del proyecto

Capas

Las capas las creamos mediante la etiqueta < div>. El anchpo del div es del ancho de la capa que lo contiene y su alto es el alto de lo que contenga el div. Eso por defecto, siempre se puede manipular mediante las etiquetas width y height

Padding y width

El padding empuja de fuera a dentro. El margin de dentro hacia afuera. El problema de padding es que se suma al width ya existente generando comportamientos no deseados. Se puede solventar de dos formas, o bien cutremente haciendo la resta entre width y padding para mantener el ancho original o bien utilizas el estilo (box-sizing: border-box) el cual hace que el padding no se sume al ancho. El border tambien se suma al width, pero el problema se solventa con el mismo estilo anterior (box-sizing: border-box).

Margin Collapsed

Sucede cuando tenemos dos elementos pegados el uno con el otro. Por defecto, sus margenes se fucionaran en uno solo, siendo que si modificamos el margen del contenido, visualmente se alterará el margen del elemento más grande que lo contiene, puesto que tienen sus márgenes colapsados. Con el estilo overflow:hidden podemos arreglar el problema del margin collapsed. Otras dos soluciones menos eficientes son el aplicar un padding o un border.

display:block, inline, inline-block

Para solventar el problema de las separaciones entre los bloques con inline-block, hay tres soluciones posibles, todas igual de poco elegantes:

  1. Eliminar el espacio entre las líneas de código, pero esto genera un código ilegible.
  2. Disminuir el tamaño de la letra de la capa contenedora y luego ir bloque por bloque interno reestableciendo el tamaño deseado.
  3. Comentar el espacio entre líneas de código,
